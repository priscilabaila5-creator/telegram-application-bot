# Короткий отчет

Сделан Telegram-бот "Заявка" без LLM и без внешних зависимостей.

Что покрыто:

- `/start` показывает приветствие и кнопку "Оставить заявку".
- Заявка собирается по шагам: имя, тип задачи, бюджет, срок, контакт.
- Заполнение заявки начинается только после кнопки "Оставить заявку".
- Тип задачи и бюджет выбираются кнопками; текст вместо кнопки мягко переспрашивается.
- Перед отправкой показывается карточка и кнопки "Отправить" / "Заполнить заново".
- "Отправить" отправляет карточку в `TARGET_CHAT_ID` и пишет "Заявка принята".
- Старая кнопка "Отправить" не отправляет пустую заявку.
- Белый список хранится в JSON и не теряется после перезапуска.
- `/adduser <id>` и `/users` доступны только стартовым админам из `ADMIN_IDS`.
- Состояние заявки хранится отдельно по `user_id`, ответы двух людей не смешиваются.
- Секретов в коде нет, настройки вынесены в переменные и `.env.example`.
- Polling-цикл продолжает работу после временной ошибки Telegram API.

Проверка:

```bash
python -m unittest discover tests
```

Для живого прогона нужны: `TELEGRAM_BOT_TOKEN`, `TARGET_CHAT_ID`, `ADMIN_IDS`.
